About Php Inspections (EA Ultimate)


7 January, 2018

After the first Php Inspections (EA Extended) announcement at Reddit, I keep collecting and working on given feedback. So far the most of it is about what will happen with Php Inspections (EA Extended), what makes Php Inspections (EA Ultimate) worth paying fot it, and of course licensing model details. Here I would like to make those three topics clear and transparent as much as possible.


Php Inspections (EA Extended) and Php Inspections (EA Ultimate)


Php Inspections (EA Extended) and Php Inspections (EA Ultimate) are two different project. Like IDEA Community Edition and IDEA Ultimate - they are evolving with own roadmaps and driven by different communities. I following the same approach in terms of roadmapping and working with communities.

Php Inspections (EA Ultimate) is based on Php Inspections (EA Extended) and bug-fixes are coming into Php Inspections (EA Extended) at first place and later getting merged into Php Inspections (EA Ultimate). This is a guaranty for further development of Php Inspections (EA Extended).

Of course, new features topic coming into play. I mentioned decoupled roadmaps, hence rules there are following:

- Code style, Language level migration, and PhpUnit-related features are coming into Php Inspections (EA Extended)
- Probable bugs, Control flow, Architecture, Security, and others are coming into Php Inspections (EA Ultimate)

Projecting to near future, projects specialization will be following:

- Php Inspections (EA Extended): foundation level code analysis tool with migration support (covers ~90% of OSS projects needs)
- Php Inspections (EA Ultimate): hi-end code analyzer (for projects with high stability, performance, maintainability and security requirements)

This should cover the current state and future development trajectory of projects. I am convinced it is a fair and open approach.


Why you might want to get a Php Inspections (EA Ultimate) license


Comparing to Php Inspections (EA Extended) features are (I'll keep updating the numbers with new releases):

- Security group: 6 new inspections, 1 enhanced inspection
- Probable bugs group: 4 new inspections, 5 enhanced inspections
- Control flow group: 3 new inspections
- Performance group: 1 new inspection

New and enhanced inspections are requiring an active license, the rest inspections are the same as in Php Inspections (EA Extended) and keeps working even without a license.

You can estimate the direction where Php Inspections (EA Ultimate) is heading and if it's your case, it would be great to get a license request from you. One more reason is that features request from our customers will be considered with needed attention: "customer is the king" approach. Or alternatively, consider buying a license as supporting Php Inspections (EA Extended) - it's challenging work happens there and it saves lots of time.


Pricing model


- OSS supporter license ($30 / 1 year): if you supported Php Inspections (EA Extended) in 2017 via Patreon, Indiegogo or PayPal. (PS: Thank you!); - Personal license (FREE / 1 year): if you are a PHP Usegroup lead (1 license per Usergroup). - Personal license ($5 / month): become a Php Inspections (EA Extended) Patron via https://www.patreon.com/kalessil in 2018 (support as long as you use the plugin). - Personal license ($50 / 1 year): you are getting support within 5 business days, but can apply onle for one license. - Commercial license ($100 / year): you are getting support within 2 business days and can apply for multiple licenses. When you buy 4 license, you are getting 5: e.g. for team of 10 people you can buy 8 licenses and will get 10 of them.
